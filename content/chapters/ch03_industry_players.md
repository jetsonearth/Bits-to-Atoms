## 这个行业里都有谁？- 玩家地图

---

你决定做一个机器人应用。打开浏览器搜了一圈，发现自己掉进了一个由几十家公司、几百个开源项目、几个生态阵营组成的丛林。每家都说自己是"平台"，每个项目都说自己是"标准"，你根本分不清谁在造机器人、谁在卖软件、谁在做工具、谁只是在蹭热度。

这一章帮你理清楚这片丛林里的势力分布。不求面面俱到 - 这个行业每个月都有新公司冒出来 - 但求让你在听到一个名字时，能快速定位它在生态里的位置。

---

## 第一层：OEM 本体厂商 - 造机器人的人

OEM（Original Equipment Manufacturer）是造机器人本体的公司。它们卖给你的是一台物理机器 - 有底盘、有手臂、有传感器、有计算平台，能动。

但"能动"和"能干活"之间有很长的距离。这就是为什么你作为开发者存在的意义。

**做四足和人形的：**

**宇树（Unitree）** 是目前对开发者最友好的中国机器人公司之一。Go2 机器狗大概一万多人民币（EDU 版），是很多开发者的第一台真机。G1 人形机器人十几万起。关键是它们的 SDK 是开源的 - unitree_sdk2 提供 Python 和 C++ 接口，支持关节级控制和高层运动指令，通过 CycloneDDS 通信。也有 ROS 2 包装和 Isaac Gym 的 RL 训练环境。对于"想买一台机器人回来写代码"这个需求，宇树目前是性价比最高的选择。

**Fourier（傅利叶）** 的 GR 系列人形机器人走开放平台路线，跑 ROS 2，用 Jetson 做机载计算。它家的背景是康复外骨骼，所以在执行器（电机 + 减速器）上有积累。

**优必选（UBTECH）** 的 Walker S 是国内最早的人形机器人之一，但开发者生态相对封闭。它更偏向企业级交付 - 你不太容易"买一台回来自己玩"，更多是"联系商务谈合作"。

**Figure AI** 和 **1X Technologies** 是美国/欧洲的人形机器人公司，融资规模很大（Figure 背后有微软、NVIDIA、OpenAI 的钱），但截至目前开发者能直接用到的 SDK 和硬件接入很有限。它们更多在内部研发和定向合作阶段。1X 倒是开源了一些数据集和仿真环境，值得关注。

**Boston Dynamics** 是行业标杆。Spot 机器狗的 SDK（spot-sdk）是目前所有机器人 SDK 里文档最好、功能最完整的 - 自主导航、机械臂操控、自定义载荷、GraphNav 路径规划、Network Compute Bridge（在机器人上跑自定义 ML 模型）。问题是 Spot 七八万美元起步，不是个人开发者随便买得起的。但如果你有机会接触一台 Spot，它的 SDK 体验会让你对"机器人开发者体验应该长什么样"建立一个很高的标准。

**银河通用（Galbot）** 是近两年崛起很快的人形机器人公司，专注商业场景落地。它的开发者生态还在建设中，值得持续关注。

**做移动底盘和 AGV 的：**

这个赛道更成熟也更碎片化。中国有快仓、海柔创新（HAI Robotics）做仓储 AGV/AMR，国外有 Locus Robotics、6 River Systems。大多数 AGV 厂商提供的是整套解决方案而不是开发平台 - 它们的"SDK"更像是一套 API 让你对接调度系统，而不是让你控制机器人底层行为。

所以... 开发者拿到手的到底是什么？这是很多新人最困惑的地方。你买了一台机器人，打开包装，然后呢？

通常你拿到的是：
- 一台物理机器，已经装好了传感器和计算平台
- 一套 SDK，让你能通过代码控制关节、读取传感器数据、发送运动指令
- 也许有 ROS 2 支持（不是所有厂商都有）
- 也许有仿真模型（URDF / USD 格式），让你在仿真环境里先测试
- 一份文档，质量参差不齐

你拿不到的是：一个能直接执行业务任务的系统。"去货架拿水瓶"，或者 "给我去麦当劳取我的外卖订单" 这些端到端的任务 - 感知、导航、抓取、调度 - 这些全部要你自己搭。OEM 给你的是原材料，你要做一桌菜。

## 第二层：中间件与平台 - 连接硬件和应用的人

如果 OEM 给你的是食材，中间件 (middleware) 就是你的厨房设备 - 锅、灶、刀具。没有它们你也能做菜（原始人烤肉不需要灶台），但效率会低到令人发指。

**ROS 2 生态** 是目前整个行业的事实标准。几乎所有主流机器人的 SDK 都提供 ROS 2 接口，几乎所有算法框架（Nav2、MoveIt、slam_toolbox）都是 ROS 2 包。你可以不喜欢 ROS 2 的某些设计决策，但你几乎绕不开它。详细内容在第 7 章展开。

**NVIDIA Isaac** 是 NVIDIA 在机器人领域的完整布局，包含好几个产品：
- **Isaac Sim**：高保真仿真平台，基于 Omniverse，能模拟真实的物理、传感器、光照。是目前 sim-to-real 工作流的主力工具
- **Isaac ROS**：GPU 加速的 ROS 2 包（品牌名叫 NITROS），专为 Jetson 优化，覆盖立体视觉、视觉 SLAM、物体检测等
- **Isaac Lab**：强化学习训练平台，能在 GPU 上并行跑几千个仿真实例

如果你用 Jetson 做计算平台（大概率你会），Isaac 生态几乎是必经之路。

**MoveIt Pro**（PickNik Robotics）是 MoveIt 2 的商业版本。PickNik 就是 MoveIt 开源项目的核心维护者。MoveIt Pro 加了可视化行为树编辑器和企业级部署工具。面向的是需要把 MoveIt 用在生产环境里的公司。

**DimOS（Dimensional OS）** 走了一条不同的路：Python-first，pip install，不强制依赖 ROS 2。开发者用 Module + Blueprint 的模式组装机器人应用，内置了 LLM agent 集成（通过 MCP 协议让大模型直接调用机器人技能）。目前支持宇树 Go2/G1、Xarm 机械臂等，还在 pre-release 阶段。利益相关声明：本书作者在这家公司工作。后面涉及 DimOS 的地方会如实描述能力和局限，但选用什么平台永远是你自己的决定。

## 第三层：工具链 - 开发者的日常装备

不管你用什么硬件、什么框架，有一些工具是几乎所有机器人开发者都会用到的。

**Foxglove** 是机器人领域的可观测性平台。你可以把它理解成"机器人的 Datadog"。它能可视化 ROS 2 话题数据（3D 点云、相机画面、TF 变换树、状态曲线），支持录制数据的回放分析，而且是 web-based 的，远程调试不需要 ssh 进机器人跑 RViz。它推出的 **MCAP** 格式正在成为 rosbag 的现代化替代。

**Rerun** 是另一个值得关注的可视化工具，开源，专门为多模态数据流设计 - 点云、图像、时间序列可以在同一个界面里同步查看。比 RViz 现代，比 Foxglove 更偏向开发者调试而非运维监控。如果你习惯了 TensorBoard 的开发体验，Rerun 会让你觉得很亲切。

**Docker** 在机器人开发中的重要性远超你的想象。ROS 2 跟 Ubuntu 版本强绑定（比如 Humble 绑 22.04，Jazzy 绑 24.04），加上各种依赖包的版本冲突，环境管理是真正的噩梦。容器化几乎是唯一的解决方案。很多团队的工作流是：仿真容器、感知容器、导航容器各跑各的，通过 ROS 2 的 DDS 通信层互相连接。

**Gazebo**（现在叫 Gz）是 ROS 生态的默认仿真器 - 比 Isaac Sim 轻量很多，适合快速原型验证和 CI 跑自动化测试。保真度不如 Isaac Sim，但启动快、资源占用小，很多团队日常开发用 Gazebo，需要高保真训练时再切 Isaac Sim。

**Weights & Biases / MLflow** 等 ML 实验追踪工具在机器人感知和 RL 训练中同样适用。如果你做过 ML 开发，这些工具你应该已经很熟了。

## 第四层：计算平台 - 机器人的心脏

**NVIDIA Jetson 系列** 是目前机器人上最主流的边缘计算平台，没有之一。

| 型号        | AI 算力       | 定位  | 适合场景            |
| --------- | ----------- | --- | --------------- |
| Orin Nano | 40-67 TOPS  | 入门  | 单模型推理、简单感知      |
| Orin NX   | 70-100 TOPS | 主力  | 多模型并行、Nav2 + 感知 |
| AGX Orin  | 最高 275 TOPS | 旗舰  | 人形机器人、复杂场景全栈    |

TOPS（Tera Operations Per Second）是 AI 推理算力的单位。一个粗略的直觉：Orin Nano 能跑一个 YOLO 模型做实时检测，AGX Orin 能同时跑视觉 SLAM + 物体检测 + 深度估计还有余量。

所有 Jetson 都跑 **JetPack SDK** - 基于 Ubuntu Linux，预装了 CUDA、TensorRT、cuDNN 和 Isaac ROS 的支持。如果你之前用过 NVIDIA 的 GPU 开发工具链，Jetson 上的体验是类似的，只是算力更有限。

## 第五层：开源社区 - 真正的基础设施

机器人行业的开源生态跟 AI/ML 社区有一个很大的不同：它更碎片化，维护者更少，很多重要的包就靠一两个人在撑。

**Open Robotics** 是 ROS 和 Gazebo 的创建者，2022 年被 Google 旗下的 Intrinsic 收购。为了确保 ROS 的社区治理不因收购而受影响，**OSRA（Open Source Robotics Alliance）** 在 Linux Foundation 下成立，负责 ROS 2 的长期维护和演进。ROS 2 的最新 LTS 版本是 Jazzy Jalisco（2024 年 5 月发布）。

**ROS-Industrial** 是一个联盟，专注于把 ROS 应用到工业制造场景。提供了一系列工业机器人（UR、FANUC、ABB、KUKA）的 ROS 2 驱动包。如果你的项目涉及工业机械臂，这是第一个要看的地方。

**LeRobot（Hugging Face）** 是最近两年冒出来的明星项目。它想做"机器人学习领域的 Hugging Face Transformers" - 统一的数据集格式、预训练策略模型（ACT、Diffusion Policy 等）、标准化的训练流水线。更重要的是它支持低成本硬件（几百块的桌面机械臂），让个人开发者也能玩真机器人学习。宇树也发布了 unitree_lerobot 做官方集成。如果你的方向是模仿学习或 RL，LeRobot 是目前最活跃的开源入口。

## "二次开发"到底是什么意思？

你会在这个行业里反复听到"二次开发"这个词。它的意思很直白：OEM 给你一台机器人（一次开发的产物），你在上面开发应用（二次开发）。

一个准确的类比：OEM 是菜市场，卖给你新鲜的食材（硬件 + 基础 SDK）。中间件是你的厨房设备（ROS 2、Isaac、MoveIt）。而你 - 二次开发者 - 是厨师。菜市场不会帮你做菜，厨房设备也不会。你得自己决定做什么菜（应用场景）、用什么菜谱（算法选择）、怎么摆盘（用户体验）。

这也是为什么"买一台机器人"和"有一个能用的机器人产品"之间，隔着巨大的工程量。那些在展会上看起来很酷的 demo，背后往往有一个团队花了几个月时间在做感知调参、导航调优、异常处理、边界情况覆盖。

对 AI 开发者来说，好消息是：二次开发中最值钱的环节 - 感知、决策、学习 - 恰好是你的强项所在。你不需要会设计电机，但你需要知道怎么在 Jetson 上高效跑一个检测模型，怎么用行为树编排一个完整的任务流程，怎么用 RL 训一个比手调参数更鲁棒的控制策略。

这张玩家地图会随时间变化 - 新公司会出现，有些会消失，生态位会重新洗牌。但分层结构是稳定的：造硬件的、做中间件的、做工具的、做芯片的、做开源的。理解了这个结构，新出现的任何玩家你都能快速归类。
